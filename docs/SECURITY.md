# Security Policy

We take the security of our projects extremely seriously. If you discover a security vulnerability within EienJS, please report it to us immediately using one of the following methods:

## 📧 Private Email (Recommended)
Send an email to our security team:
- **Email:** luffynando@gmail.com

Please include as much detail as possible about the vulnerability, including:
- A clear description of the issue
- Steps to reproduce the vulnerability
- Any relevant code snippets or logs
- Potential impact of the vulnerability

## 📝 GitHub Security Advisory
Alternatively, you can open a private security advisory through GitHub:
1. Go to the **Security** tab of this repository
2. Click on **Advisories**
3. Click **New draft advisory**
4. Fill in the details and submit

## What to Expect
- All security reports will be treated with the highest priority
- We will acknowledge receipt of your report within 24 hours
- We will work quickly to investigate and fix the vulnerability
- We will credit you for the discovery in accordance with responsible disclosure guidelines
- We will keep you updated on our progress

## Responsible Disclosure Guidelines
Please follow these guidelines when reporting a vulnerability:
- Do not publicly disclose the vulnerability until we have had a chance to fix it
- Do not exploit the vulnerability in any way
- Do not share the vulnerability details with anyone else
- Provide us with enough information to reproduce and fix the issue

We appreciate your help in keeping our projects secure!
